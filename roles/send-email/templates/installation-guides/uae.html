Instructions to update MC CATALOGUE from version 5.09.03 to version 5.09.04

Note: this update cannot be used for older versions than 5.04.18

TEST & TRAINING ENVIRONMENT

0. Check file system free space:

Server  mcc-test.uaencb.ae
Using PUTTY login to server  mcc-test.uaencb.ae (192.168.10.55) as user mcc
Execute command:
df -h
Verify free space for the following file systems:

 / (free space should be more than 5GB)
/aura (free space should be more than 50GB)
/export/home (free space should be more than 10GB - you can free some space by deleting old installation sources in  /export/home/mcc/install- sources)

 


1. Download and unzip file from AURA file server to some directory on MCC application server ( mcc-test.uaencb.ae)

Using WINSCP login to server  mcc-test.uaencb.ae ( 192.168.10.55) as user mcc
Unzip file mcc50904-aest.zip into  /export/home/mcc/install- sources/ 2018-10-16_5.09.04-AE- update (verify that file /export/home/mcc/install- sources/2018-10-16_5.09.04-AE- update/install/setup exists)


2.Execute SETUP: 

Using PUTTY login to server  mcc-test.uaencb.ae (192.168.10.55) as user mcc

To start the setup execute command:

sh /export/home/mcc/install- sources/ 2018-10-16_5.09.04-AE- update /install/setup

Follow all instructions provided by the setup:


2A. Choose which application to update:

MCC SETUP
----------------

Enter number 1 to update MCC in folder: /aura /mcc-test/aura-skelet
Enter number 2 to update MCC in folder: /aura/mcc-schilda/aura-skelet
Enter number 3 to update MCC in folder: /aura/mcc-train/aura-skelet
Enter number 4 to update MCC in folder: /aura/mcc-tir/aura-skelet
 or enter full path to AURA_HOME (the path where you want to update MCC - for example /aura/mcc-test/aura-skelet):

To update for example MCC Test application:
Enter number 1and press Enter

2B. Email configuration

Setup will ask you for email configuration:

Enter "no" to disable this feature.


2C. Stop MCC if running
If MCC is running, setup will ask you for stopping it:

MCC is running and must be stopped before starting the update, would you like to stop it? Y[es]/N[o]

Press "y" to disable and stop MC CATALOGUE

2D. BACKUP
 MCC is stopped
In case of updating production system, we recommend to perform backup of the fo llowing now:
 - MCC database
 - MCC workspaces in  /aura/mcc-test/aura-skelet/ mcc/workspaces (you can exclude  /aura/mcc-test/aura-skelet/ mcc/workspaces/var/nmcrlindex)
Would you like start MCC update process for MCC installation in  /aura /mcc-test/aura-skelet ? Y[es]/N[o]

We can skip database and workspace backup for Test & Training environment.
Enter "y" to continue the update.

2E. MCC UPDATE process

Setup will automatically update:

  • MCC Skeleton
  • Apache Tomcats
  • JAVA
  • mcc.yaml configuration file
  • MCC war and jar files
  • MCC database

After the successful update you will get the following message:

MCC Update process has finished
Before starting MCC we recommend to check values in mcc.yaml file located in:
/aura/mcc-test/aura-skelet/ mcc /conf/mcc.yaml

Would you like to start MCC now? Y[es]/N[o]

Press "Y"

3. MC CATALOGUE URL Check

Open the following URL to check if all modules and services are running after the update:

MCC TEST application:  https://mcc-test.uaencb.ae/ mcc/faces/public/ ApplicationCheck.xhtml
MCC SCHILDA application:  https://mcc-schilda.uaencb.ae/ mcc/faces/public/ ApplicationCheck.xhtml
MCC TIR application:  https://mcc-tir.uaencb.ae/mcc/ faces/public/ApplicationCheck. xhtml
MCC TRAIN application:  https://mcc-train.uaencb.ae/ mcc/faces/public/ ApplicationCheck.xhtml
